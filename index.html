Knockknock

An alternative smartphone interface

Download this project as a .zip file Download this project as a tar.gz file

Idea

We want to make an alternate interface for controlling smartphones. It's annoying having to take the phone out of the pocket for every little thing. We want to make it possible to perform simple predetermined actions on the phone by simply tapping the phone while it's in the pocket.

TEST

Use-cases where it's nice to not have to take out the phone:

  • While in meeting: silencing a ringing phone or sending a predetermined message
  • Start recording sound
  • Control music apps
  • Control a stop watch
  • For any use when it's inappropriate to be using the phone

Goal

At the end of this project we want an interface which is reliable enough for everyday use and we will have two example applications using this interface. The project must be done and ready for presentation by March 16. when at least one example app will be presented to our class.

The first app will be a distance measuring app for golf, two taps for marking the start GPS location for where the shot was taken from, and then three taps for marking where the ball landed. The app will then read back to the user the distance the ball has traveled.

The second app will be an app for controlling media-playback in smartphones. It will be possible to stop, pause or go to next song by tapping the phone. Should ideally work for most music playing applications like the built in player, Spotify, Pandora etc.

Main technical challenges

  • Properly reading user input
  • Giving feedback to the user
  • Making this happen while the phone is in a locked state
  • Creating an interface which is reliable enough

Solutions to some of the technical challenges

  • Fusing accelerometer and sound recording to detect the signature of a tap to the side of the phone (maybe also gyroscope)
  • Launching the app as a service to make it run in background and during lock state
  • Feedback via text-to-speech, vibration or simple sounds

Progress

The project has been presented to the class and our professor has accepted it.

Homepage is up

We have determined some of the most important challenges and modules in our project and divided them between us.

Programmed several simple apps for testing and getting used to: accelerometer, microphone, vibration motor, text-to-speech, controlling media apps etc.

Started looking at what kind of data we can get from the accelerometer and microphone and ways to analyze these.
